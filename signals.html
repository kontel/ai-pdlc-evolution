01 Ambient intent capture
Apr 23
Microsoft teases an “agentic, ambient” next OS
Microsoft sketched its next-generation Windows direction this week: an OS that semantically understands what is on your screen at all times as a first-class capability. The framing is explicit — agentic, ambient, multimodal — and it positions ambient intent capture as platform-level, not application-level. If this lands, every app becomes a potential intent source rather than the destination.

OpenAI’s “Ambient” screenless device coverage intensifies
Coverage of OpenAI’s rumoured screenless device crystallised into a coherent architectural picture this week: LLM-as-kernel instead of files-and-apps; intent flowing in via voice and sensors; “Agentic Workspaces” created on demand; results returned via voice. If accurate, it is the first mainstream consumer device explicitly designed around the three-layer model.

DigitalOcean — Ambient agents: the next frontier in context-aware AI
A clean primer that locks down the four properties separating ambient agents from chatbots: continuous (not episodic), environmental (not device-bound), proactive (acting on their own), multimodal (across sensors, voice, vision, logs). Useful as a working definition when arguing internally about whether something is “just a chatbot” or genuinely ambient.

The PDLC analogue is direct: ambient capture of what the team is actually deciding in meetings could pre-fill the spec, the JTBD, and the agent context window without anyone typing it twice.

02 Agentic execution
Apr 16
OpenAI — “Codex for (almost) everything”
The biggest release of the fortnight. Codex now does desktop computer use (sees, clicks, types across any Mac/Windows app), ships an in-app browser, exposes 90+ plugins (Atlassian Rovo, GitLab, CircleCI, Microsoft Suite, Render, etc.), and can schedule itself to wake up days or weeks later and resume long-running work. Distributed to ~3M weekly developers in one update. This single move closes most of the “but the agent can’t actually do things” gap.

Useful detail beyond OpenAI’s own post: multiple agents run on your Mac in parallel without interfering with what you’re doing in other apps. That parallelism is the bit that flips the mental model — agents stop being “a tab you switch to” and become “background processes you tune in to”.

Enterprise browser agents (Comet Enterprise, Island AI Browser)

03 Ephemeral verification UX
Apr 24
Google — A2UI v0.9 generative-UI spec
Google released A2UI v0.9 this week. It is declarative (not executable code), framework-agnostic (React, Flutter, Lit, Angular renderers shipped), and explicitly safe-by-construction: agents can only request components from a pre-approved catalog the host app maintains. This is the first credible standard for “agents emit UI on demand” that an enterprise security team can sign off on.
StandardGoogleGenerative UI
Apr 22
CopilotKit — building with A2UI + AG-UI
A complementary piece showing the natural pairing: A2UI describes what the agent wants to render; AG-UI streams the live execution. Together they form an end-to-end ephemeral-UI toolchain — declarative surface + streaming state — that matches the workspace prototype’s Flow mode almost exactly.

The New Stack — Agent UI standards multiply
A useful market read: MCP Apps and A2UI are converging on the same idea from different directions. UIs become declarative artifacts agents emit, not products engineers ship. Expect a brief standards war and then de-facto interop — bet on the abstraction, not the specific spec.

EverydayUX — designing for disposable UI
A clear thesis for design teams: when the agent generates the screen, designers stop owning “screens” and start owning component catalogs, decision frames, and trust signals. The artifact shifts from a Figma file to a system spec. This is the design-side mirror of the engineering “harness, not code” shift the writeup already covers in Section 5.

How this page is produced

Each Sunday evening, the refresh-signals Claude Code skill (in .claude/skills/refresh-signals/) runs the same scan we ran for the seed entries:

  1. Scope. Last 7–14 days only. Strictly enforced — older items belong in the writeup, not here.
  2. Topics. Agentic PDLC, the three layers (ambient / agentic / ephemeral), product engineer role, harness engineering, trust calibration, multi-agent orchestration, generative UI standards.
  3. Per signal. 2–5 sentences. One source link in the title. Plain English. Why-it-matters first, what-it-is second.
  4. Layering. Each item is filed under one of three layers, or the cross-cutting rail (orchestration patterns, role evolution, regulation). Five items per layer per week is plenty — signal beats volume.
  5. Append, don’t overwrite. Each new week is added as a new <section class="sig-week"> at the top; older weeks stay below for longitudinal reading.

The skill produces a draft block that the PE reviews before commit — this is a Layer 3 surface in its own right. To run it: invoke the skill from Claude Code on this repo and follow the prompts.
